Phase 2 — cleanup and removal

Goal: remove wrong structure before rebuilding.

This is where you clean out old decisions.

Delete or refactor

old CloudPano integration

unused viewer components

any old storage buckets you no longer want

dead Supabase tables/functions/policies

old upload flows that bypass your final logic

duplicate auth wrappers

frontend-only billing logic pretending to be secure

unused API routes

abandoned experimental pages

broken analytics experiments

any mixed “tour/project/property” naming chaos

Rename for consistency

Use one clean naming system.

Recommended:

property = main listing/experience

property_media = gallery/panorama files

subscription = billing state

plan = package definition

Avoid mixing:

tour

project

listing

space

property

Pick one main model and stick to it.
My recommendation: property.

Output

A cleaner repo and database with less confusion.

Whenever you feel tempted to jump around, ask:
Is this P0, P1, P2, or P3?

If it is not P0 or P1 right now, it probably waits.

What is missing before implementation starts

Before coding, lock these decisions today:

exact plan limits

exact grace period behavior

whether 2D and 360 always live on same page

which plans get embeds

which plans get lead capture

exact upload size limits

exact branding rules per plan

Do not leave those vague.

My blunt recommendation on where to start right now

Start with this exact order:

1. Audit current codebase and Supabase

Because you already said some things were built in the wrong place.

2. Freeze the architecture

Because without that, cleanup becomes random.

3. Rebuild database and backend foundation

Because that controls everything else.

4. Reconnect frontend after the core is clean

Because UI on top of messy logic is fake progress.
